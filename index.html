In this lesson we will learn the following

  • How to create links to pages on other websites on the internet
    • Click Me
  • How to create link to other pages on your own websites
  • The difference between absolute and realtive links
  • How to display an image on a webpage using HTML

Absolute links are links to content on another page, they will always contain the protocol://domain/path format/

Relative links are links to pages within our own website, like to go from menu to about us pages

This works as long as the pages being traversed are within the same directory, if not then you need to add to the link.

Images

You can add images to the page with an <img> tag. These are self closing tags, that work in the same manor as relative and absolute links. You can also add alt text to an image for when the image doesn't load.

Commit messages

In this lesson we learn about the following

  1. How to write meaningful commit messages
  2. Why meaningful messages are important
  3. When to Commit

    4. Yes, commit messages are important enough to warrent their own lesson. Those reasons are

    • When applying for jobs employers will look through your commits, this could help you stand out, especially early in oyur career
    • With a good commit history the team your are working with will be able to see where bugs have poped up
    • It will also allow you to get back to work quicly on projects that have been on the shelf for a while.

    When you commit is also important for reason like:

    • Commiting is like taking a snapshot of you code at that time, allowing you to revert back when a bug happens, or when trying something new doesn't work the way you thought.
    • It is considered best practice to make a commit with every meaningful code change. This could mean anything from a bug, typo correction to even you have gained experiance and improved the code of the page.
    • It is even a good idea to commit when things are finally working the way they are suppose to. This way when it does break you at least have a reference to time it worked properly.
